# 誤字・脱字チェック: $ARGUMENTS

## 概要

指定されたファイルまたはディレクトリ内のMarkdownファイルを丁寧にチェックし、誤字・脱字を検出・報告する。

## 対象

チェック対象: $ARGUMENTS

- ファイルパスが指定された場合：そのファイルをチェック
- ディレクトリパスが指定された場合：配下の全`.md`ファイルをチェック
- 引数なしの場合：現在開いているファイルまたはユーザーに確認

## チェック項目

### 1. 日本語の誤字・脱字

- 漢字の誤変換（例：「機能」→「気能」）
- 送り仮名の誤り（例：「行なう」→「行う」）
- 同音異義語の誤用（例：「以外」と「意外」）
- ひらがな・カタカナの誤り
- 句読点の不統一（「、。」と「，．」の混在）

### 2. 技術用語の表記揺れ

- カタカナ表記の統一（例：「サーバー」vs「サーバ」）
- 英語表記の統一（例：「JavaScript」vs「Javascript」）
- 略語の統一（例：「API」vs「Api」）

### 3. Markdown構文エラー

- 見出しレベルの飛び（h1 → h3）
- リンク構文の誤り
- コードブロックの閉じ忘れ
- 表の構文エラー

### 4. 文章の整合性

- 主語と述語の不一致
- 文末表現の不統一（「です・ます」と「だ・である」の混在）
- 箇条書きの形式不統一

## 出力形式

```markdown
## チェック結果: [ファイル名]

### 検出された問題

| 行 | 種類 | 問題箇所 | 修正提案 |
|----|------|----------|----------|
| 10 | 誤字 | 「機能」→「気能」 | 「機能」に修正 |
| 25 | 表記揺れ | 「サーバー」と「サーバ」混在 | 「サーバー」に統一 |

### 問題なし
- [問題がなかった項目]

### サマリー
- チェックファイル数: X
- 検出問題数: Y
- 重要度: 高/中/低
```

## 実行手順

1. 対象ファイルを全て読み込む
2. 各チェック項目を順番に確認
3. 問題を検出したら行番号と修正提案を記録
4. 結果をMarkdown形式で報告
5. 修正が必要な場合、ユーザーに確認の上で修正を実行

## オプション

- `--fix`: 検出した問題を自動修正（確認あり）
- `--strict`: 厳格モード（警告レベルの問題も報告）
- `--ignore-code`: コードブロック内をスキップ

## 使用例

```bash
# 単一ファイルをチェック
/typo-check rules/cc-best-practice-compliance.md

# ディレクトリ内の全ファイルをチェック
/typo-check rules/cc-bp/

# 厳格モードでチェック
/typo-check rules/ --strict

# 自動修正付きでチェック
/typo-check CLAUDE.md --fix
```

## 注意事項

- コードブロック内の内容は原則としてチェック対象外
- 英語のみのファイルは日本語チェックをスキップ
- 固有名詞や意図的な表記は誤検出の可能性あり
- 修正提案は参考であり、最終判断はユーザーに委ねる
