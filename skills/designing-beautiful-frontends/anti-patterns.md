# アンチパターン（絶対に避ける）

このドキュメントでは、「AIスロップ」と呼ばれる汎用的で退屈なデザインパターンを列挙する。これらを避けることで、記憶に残るUIを構築できる。

---

## 1. 使い古されたフォント

### 問題のあるフォント

| フォント | 問題点 |
|---------|-------|
| **Inter** | SaaS/スタートアップで多用されすぎ。一目で「汎用テンプレ」感 |
| **Roboto** | Googleのデフォルト。Androidアプリ感 |
| **Arial** | Windows 95からの遺産。古臭い |
| **Helvetica** | 使い古されて差別化不可能 |
| **Open Sans** | 無個性。どこにでもある |
| **Lato** | 過剰に使用されている |
| **system-ui** | デフォルト感。意図のなさを示す |

### なぜ問題か

これらのフォントは「安全」であるがゆえに、世界中の何百万ものサイトで使われている。結果として：
- ユーザーの記憶に残らない
- ブランドの差別化ができない
- 「テンプレートを使った」という印象を与える

### 解決策

→ [typography-guide.md](typography-guide.md) の推奨フォントを使用

## 2. SaaS汎用デザイン

### 典型的なパターン

```
❌ 避けるべき組み合わせ:
- 紫グラデーション（#6366f1 → #a855f7）+ 白背景
- 丸みを帯びたカード（border-radius: 16px）
- 青いCTAボタン（#3b82f6）
- 左右対称のセクション配置
- ヒーロー → 3カラム特徴 → テスティモニアル → CTA → フッター
```

### なぜ問題か

この組み合わせは「SaaSテンプレート」として広く使われており、見た瞬間に：
- 「どこかで見た」という印象
- 「本気で作っていない」という印象
- 「競合と同じ」という印象

### 解決策

1. **色**: 紫グラデーションを避け、プロジェクト固有の色を選ぶ
2. **レイアウト**: 非対称、重なり、グリッド破りを取り入れる
3. **角丸**: 全部同じにしない。0, 4px, 50% など意図的にバリエーションを

## 3. 意味のないアニメーション

### 問題のあるパターン

```tsx
// ❌ すべての要素に同じフェードイン
<motion.div
  initial={{ opacity: 0 }}
  animate={{ opacity: 1 }}
>
  {/* すべての要素がこれ */}
</motion.div>

// ❌ 過度なパララックス
// スクロールするたびにすべてが動く

// ❌ 無意味なホバーエフェクト
// 何もアクションがないのにホバーで変化
```

### なぜ問題か

- **パフォーマンス低下**: 不要なアニメーションはCPU/GPUを消費
- **ユーザー体験悪化**: 動きすぎは疲れる、集中を妨げる
- **安っぽさ**: 「とりあえずアニメーション」は素人感

### 解決策

**高インパクトな瞬間のみアニメーション**:
1. ページロード時のスタガードアニメーション（1回だけ）
2. 重要なCTAのホバー
3. 状態変化のフィードバック（送信成功など）

```tsx
// ✅ 意図的なスタガードアニメーション
const containerVariants = {
  hidden: { opacity: 0 },
  visible: {
    opacity: 1,
    transition: { staggerChildren: 0.1 },
  },
};
```

## 4. Bootstrap/Materialのデフォルト

### 問題のあるパターン

- Bootstrap Blue（#0d6efd）
- Material Purple（#673ab7）
- デフォルトのボタンスタイル
- デフォルトのフォームスタイル
- デフォルトのカードシャドウ

### なぜ問題か

一目でフレームワークが分かる = 手抜きの印象

### 解決策

フレームワークを使う場合は必ずカスタマイズ：

```css
/* ✅ カスタムプロパティで上書き */
:root {
  --primary: #your-brand-color;
  --radius: 0; /* または独自の値 */
}
```

```js
// ✅ Tailwindの場合、theme.extendで拡張
module.exports = {
  theme: {
    extend: {
      colors: {
        primary: '#your-brand-color',
      },
    },
  },
};
```

## 5. 予測可能すぎるレイアウト

### 問題のあるパターン

```
❌ テンプレートレイアウト:

┌─────────────────────────────┐
│          ヘッダー           │
├─────────────────────────────┤
│                             │
│     ヒーローセクション       │
│                             │
├─────────────────────────────┤
│  特徴1  │  特徴2  │  特徴3  │  ← 常に3カラム
├─────────────────────────────┤
│      テスティモニアル        │
├─────────────────────────────┤
│          CTA                │
├─────────────────────────────┤
│         フッター            │
└─────────────────────────────┘
```

### なぜ問題か

- 見た瞬間に「テンプレート」と分かる
- スクロールの予測がつく = 退屈
- 差別化ができない

### 解決策

**グリッドを破る要素を追加**:

```
✅ 予想外のレイアウト:

┌─────────────────────────────┐
│     ヘッダー（固定）         │
├─────────────────────────────┤
│                   ┌─────────┤
│  超大きなタイトル  │ 装飾要素 │  ← 重なり
│                   └─────────┤
├──────────┬──────────────────┤
│ サイドバー│                  │
│（スクロ │   コンテンツ      │  ← 非対称
│  ール固定）│                  │
├──────────┴──────────────────┤
│    斜めに配置された要素      │  ← 対角線フロー
└─────────────────────────────┘
```

## 6. 中途半端なカスタマイズ

### 問題のあるパターン

```css
/* ❌ ほぼデフォルトのまま */
.button {
  background: #3b82f6; /* Tailwind blue-500 */
  padding: 8px 16px;
  border-radius: 4px;
}

/* ❌ 控えめすぎるカスタマイズ */
h1 {
  font-size: 2rem;
  font-weight: 600;
}
```

### なぜ問題か

中途半端なカスタマイズは「できなかった」ように見える

### 解決策

**極端に振る**:

```css
/* ✅ 大胆なスタイリング */
.button {
  background: #000;
  padding: 16px 32px;
  border: 4px solid #000;
  box-shadow: 4px 4px 0 #ff3366;
}

h1 {
  font-size: 6rem;
  font-weight: 900;
  letter-spacing: -0.03em;
  line-height: 0.9;
}
```

## 7. アクセシビリティ無視のデザイン

### 問題のあるパターン

- コントラスト比が低すぎるテキスト
- フォーカス状態の欠如
- クリック領域が小さすぎるボタン
- prefers-reduced-motion 無視

### なぜ問題か

- 法的リスク（WCAG準拠が求められる場合）
- ユーザー体験の悪化
- SEOへの悪影響

### 解決策

```css
/* ✅ アクセシビリティ対応 */
@media (prefers-reduced-motion: reduce) {
  * {
    animation: none !important;
    transition: none !important;
  }
}

/* フォーカス状態 */
:focus-visible {
  outline: 2px solid var(--accent);
  outline-offset: 2px;
}

/* 十分なクリック領域 */
.button {
  min-height: 44px;
  min-width: 44px;
}
```

## チェックリスト

UI実装時に以下を確認：

- [ ] Inter/Roboto/Arialを使っていない
- [ ] 紫グラデーション + 白背景を使っていない
- [ ] Bootstrapのデフォルト色を使っていない
- [ ] 全要素に同じアニメーションを適用していない
- [ ] レイアウトに予想外の要素がある
- [ ] カスタマイズが「極端」である（中途半端でない）
- [ ] アクセシビリティを考慮している
