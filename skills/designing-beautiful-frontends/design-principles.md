# デザイン原則

## 1. Purpose（目的）の明確化

### 質問リスト
- このインターフェースが解決する問題は何か？
- ユーザーは誰か？（年齢層、技術レベル、文化的背景）
- ユーザーが達成したいゴールは何か？
- このUIを通じてユーザーはどんな感情を持つべきか？

### 感情マッピング例

| 目的 | 狙う感情 | デザインアプローチ |
|------|---------|-------------------|
| 購買促進 | 興奮、緊急性 | 鮮やかな色、動き、大胆なCTA |
| 信頼構築 | 安心、プロフェッショナル | 落ち着いた色、余白、整然としたレイアウト |
| 創造性刺激 | 驚き、インスピレーション | 予想外のレイアウト、実験的タイポグラフィ |
| 効率重視 | 明快、コントロール感 | 高コントラスト、明確な階層、即座のフィードバック |

## 2. Tone（トーン）の選択

### 美学スタイル詳細

#### ブルータルミニマリズム
```
特徴:
- 極端なシンプルさ
- 大胆な空白（ページの50%以上）
- モノクロまたは2色のみ
- 装飾の完全排除

適したプロジェクト:
- アーティストポートフォリオ
- 建築事務所
- ハイエンドファッション

参考: 無印良品、Apple初期

実装のコツ:
- font-weight: 100 または 900 のみ使用
- padding/margin は 4rem 以上を多用
- ボーダーは 1px solid black のみ
```

#### ネオブルータリズム
```
特徴:
- 太いボーダー（3-8px）
- 強いドロップシャドウ（offset明確）
- 高彩度の原色
- 生々しいHTML感

適したプロジェクト:
- テックスタートアップ
- クリエイティブツール
- 若者向けサービス

参考: Figma、Notion（一部）、Gumroad

実装のコツ:
- box-shadow: 4px 4px 0 black
- border: 3px solid black
- 背景に鮮やかな単色
```

#### マキシマリストカオス
```
特徴:
- 豊かなレイヤー
- 重なる要素
- 鮮やかな色彩の混在
- 意図的な「乱雑さ」

適したプロジェクト:
- 音楽フェスティバル
- アートギャラリー
- 実験的ブランド

参考: デヴィッド・カーソン、90年代Webデザイン

実装のコツ:
- position: absolute を多用
- transform: rotate() でランダム感
- mix-blend-mode で色の重なり
```

#### レトロフューチャリズム
```
特徴:
- ノスタルジーと未来感の融合
- グラデーションメッシュ
- ネオン効果
- 80年代/90年代のリバイバル

適したプロジェクト:
- ゲーム
- 音楽系
- エンターテイメント

参考: Synthwave、Cyberpunk

実装のコツ:
- グローエフェクト（box-shadow多重）
- グリッドライン背景
- アニメーションするグラデーション
```

#### オーガニックフロー
```
特徴:
- 自然な曲線
- 有機的なフォルム
- 柔らかいグラデーション
- 自然からのインスピレーション

適したプロジェクト:
- ウェルネス/健康
- 食品/飲料
- 環境系

参考: Headspace、Calm

実装のコツ:
- border-radius: 50% や不規則な値
- SVGのblob形状
- 淡いアースカラー
```

#### エディトリアル
```
特徴:
- 雑誌的レイアウト
- タイポグラフィ重視
- コラム構成
- 写真との調和

適したプロジェクト:
- メディア/出版
- ファッション
- 文化系

参考: NYTimes、Vogue

実装のコツ:
- CSS Grid で複雑なコラム
- 大きなドロップキャップ
- 引用の特別なスタイリング
```

## 3. Constraints（制約）の把握

### 技術的制約チェックリスト
- [ ] ブラウザサポート範囲（IE11必要？）
- [ ] パフォーマンス要件（LCP、CLS目標）
- [ ] アクセシビリティ基準（WCAG 2.1 AA？）
- [ ] レスポンシブ要件（モバイルファースト？）
- [ ] アニメーション制限（prefers-reduced-motion対応？）

### ビジネス制約チェックリスト
- [ ] 既存ブランドガイドライン
- [ ] 既存デザインシステム
- [ ] 開発期間/リソース
- [ ] チームのスキルセット

## 4. Differentiation（差別化）

### 記憶に残る要素の作り方

#### レベル1: 独自のマイクロインタラクション
- ボタンホバー時に予想外の変化
- スクロール時のパララックス効果
- カーソルのカスタマイズ

#### レベル2: 予想外のレイアウト
- グリッドを意図的に破る
- 重なり合う要素
- 非対称な配置

#### レベル3: 特徴的なビジュアル要素
- カスタムイラスト
- 独自のアイコンシステム
- ブランド固有のパターン

#### レベル4: インタラクティブな体験
- スクロールストーリーテリング
- ゲーミフィケーション要素
- 音や動きへの反応

### 差別化の質問
> 「このサイトを見た人が、3日後に思い出せる要素は何か？」

この質問に答えられないなら、差別化が足りない。
